# АІС «Навчальні тести» (TestifyApp)

Система для створення та проходження навчальних тестів. Додаток дозволяє зареєстрованим користувачам створювати публічні або приватні тести, а гостям — проходити їх за унікальним invite-кодом.

## 🚀 Основний функціонал

- **Для авторів:**
    - Створення тестів із гнучкими налаштуваннями (публічний/приватний, ліміт часу, показ правильних відповідей).
    - Зручний конструктор запитань та варіантів відповідей.
    - Детальна статистика по кожному тесту (результати всіх проходжень учасниками).
    - Генерація унікальних invite-посилань для приватних тестів.

- **Для користувачів та гостей:**
    - Проходження публічних тестів із загального каталогу.
    - Гостьовий доступ до приватних тестів (достатньо вказати лише ім'я та прізвище).
    - Таймер проходження (якщо встановлено автором).
    - Миттєвий перегляд результатів (набрані бали та розбір правильних/неправильних відповідей).

## 🛠 Технологічний стек

**Frontend:**
- React 18 + TypeScript
- Vite
- React Router v6
- TanStack Query (React Query)
- Axios
- Tailwind CSS v4

**Backend:**
- Node.js (v22 LTS) + Express + TypeScript
- Knex.js (Query Builder)
- PostgreSQL 16
- JWT (jsonwebtoken) + bcrypt (авторизація та хешування)

**Інфраструктура:**
- Docker та Docker Compose
- Nginx (Reverse Proxy)

## ⚙️ Встановлення та запуск (Локальне середовище)

### 1. Попередні вимоги
- Встановлений [Node.js](https://nodejs.org/) (рекомендовано v22 LTS)
- Встановлений та запущений [Docker Desktop](https://www.docker.com/)

### 2. Налаштування змінних середовища
Створіть файл `.env` у кореневій папці проекту та додайте наступні налаштування:

```env
# База даних PostgreSQL
POSTGRES_DB=testify_db
POSTGRES_USER=testify_user
POSTGRES_PASSWORD=secret_postgres_password

# Порт для БД на хост-машині (щоб уникнути конфлікту зі встановленим Postgres)
DB_LOCAL_PORT=5433

# Секретний ключ для шифрування JWT токенів
JWT_SECRET=your_super_secret_jwt_key_here
```

### 3. Запуск інфраструктури (База даних)
Щоб запустити базу даних через Docker, виконайте команду в корені проекту:

```bash
docker compose up -d db
```

### 4. Налаштування Backend
Перейдіть у папку бекенду, встановіть залежності та запустіть міграції бази даних:

```bash
cd backend
npm install
npm run migrate:up
```

Запустіть сервер для розробки (працюватиме на порту 5000):
```bash
npm run dev
```

### 5. Налаштування Frontend
Відкрийте новий термінал, перейдіть у папку фронтенду та встановіть залежності:

```bash
cd frontend
npm install
```

Запустіть клієнтський додаток (працюватиме на порту 5173):
```bash
npm run dev
```

## 🐳 Запуск через Docker Compose (Повне розгортання)

Для запуску всієї системи (База даних, API, Фронтенд та Nginx) у контейнерах, використовуйте команду з кореня проекту:

```bash
docker compose up -d
```
Після цього додаток буде доступний за адресою `http://localhost`.
